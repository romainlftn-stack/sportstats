# Product

## Register

product

## Users

Solo user (personal): an athlete managing a running-rehab protocol (ITBS). Uses the tool daily or weekly to review progress, track sessions, and adjust the training schedule. Context: mobile (on-the-go check-ins) and desktop (weekly review). Always knows what the data means — no explanation needed.

## Product Purpose

Personal training dashboard that surfaces session completion, running volume, ITBS pain score (EVA), and protocol progress at a glance. Success = the user can open the app, understand where they stand in under 5 seconds, and act (reschedule, mark done, check trends).

## Brand Personality

Minimal · quiet · deliberate. The interface recedes; the numbers speak.

## Anti-references

- Strava, Apple Fitness rings: gamification, celebratory color bursts, social metrics
- Generic consumer wellness apps: rounded pastel cards, emoji everywhere, motivational copy
- Flashy SaaS dashboards: gradient backgrounds, hero metrics, "100K users" stats

## Design Principles

1. **Numbers first.** Every visual decision should make data faster to read, not more beautiful to look at.
2. **Signal over decoration.** Color carries meaning (green = good, red = stop, orange = warning). Never decorative.
3. **The interface disappears.** At its best, the user sees the data, not the app.
4. **Earn every interaction.** Animations exist to communicate state change, not to impress.
5. **One user, full trust.** No onboarding, no empty-state handholding. Assume mastery.

## Accessibility & Inclusion

WCAG AA. Dark mode fully supported (system preference + manual override). Reduced motion respected.
